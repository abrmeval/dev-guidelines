The following documents are required for every project to ensure consistency and clarirty. 
All documents except for **README** files and AI documents, should be placed in a `docs` folder in the root directory of the project, and should follow the naming conventions outlined in the NAMING_CONVENTIONS.md file.

- README files should be placed in the root directory of each project or subdirectory, and should follow the structure outlined in the STRUCTURE.md file.
- AI documents should be placed in an `AI` folder in the root directory of the project

Required Documents:
- **README.md** - A README file providing an overview of the project, its structure, and instructions for getting started.
- **CHANGELOG.md** - A changelog file documenting all notable changes made to the project, following the format outlined in the CHANGELOG.md template.
- **CONTRIBUTING.md** - A contributing guide outlining how to contribute to the project, including guidelines for code style, testing, and submitting pull requests.
- **/docs/ARCHITECTURE.md** - A document describing the architectural decisions and patterns used in the project, including diagrams and explanations of how different components interact.
- **/docs/DEPENDENCIES.md** - A document listing all external dependencies used in the project, including their versions and purposes.
- **/docs/STACK.md** - A document describing the technology stack used in the project, including programming languages, frameworks, libraries, and tools.
- **/docs/PROJECT_STRUCTURE.md** - A document outlining the project structure, including directories and files, and their purposes.
- **/docs/EXTERNAL_SERVICES.md** - A document listing any external services or APIs the project interacts with, including their purposes and how to set them up.
- **/docs/TESTING_TOOLS.md** - A document describing the testing tools and frameworks used in the project, including instructions for running tests and interpreting results.
- **/docs/INFRASTRUCTURE/WORKFLOWS.md** - A document describing the CI/CD workflows set up for the project, including build, test, and deployment processes.
- **/docs/INFRASTRUCTURE/CLOUD_RESOURCES.md** - A document listing any cloud resources used by the project, including their purposes and how to set them up.
- **/docs/INFRASTRUCTURE/DEPLOYMENT.md** - A document describing the deployment process for the project, including any scripts or tools used, and instructions for deploying to different environments.
- **/docs/TECH_PLANNING/** - A folder containing any technical planning documents for the project, such as design docs, architecture decision records, functional specifications, and technical specifications.
- **/docs/BUSINESS/** - A folder containing any business-related documents for the project, such as business requirements, business rules, and overview of functional requirements and other related documentation.
