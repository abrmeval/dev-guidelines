# Naming Conventions for Projects in .NET and Node.js

.NET Projects should follow the standard .NET naming conventions, which include:
- Projects: PascalCase (e.g., `CompanyName.ProjectName.TypeName`) eg. `MyCompany.MyProject.WebApi`
- Namespaces: PascalCase (e.g., `CompanyName.ProjectName.TypeName.NamespaceName`) eg. `MyCompany.MyProject.WebApi.Services`
- Assemblies: PascalCase (e.g., `CompanyName.ProjectName.TypeName`) eg. `MyCompany.MyProject.WebApi.dll`
- Files: PascalCase (e.g., `TypeName.cs`) eg. `MyClass.cs`

Node.js Projects should follow the standard Node.js naming conventions, which include:
- Projects: kebab-case (e.g., `project-name`) eg. `my-project
- Modules: camelCase (e.g., `moduleName`) eg. `myModule`
- Files: kebab-case (e.g., `file-name.js`) eg. `my-file.js`
- Directories: kebab-case (e.g., `directory-name`) eg. `my-directory`
