C# language naming conventions should be followed for all code in this repository. This includes:
- Classes: PascalCase (e.g., `MyClass`)
- Methods: PascalCase (e.g., `MyMethod()`)
- Variables: camelCase (e.g., `myVariable`)
- Constants: PascalCase (e.g., `MyConstant`)
- Interfaces: PascalCase with an "I" prefix (e.g., `IMyInterface`)
- Enums: PascalCase (e.g., `MyEnum`)
- Namespaces: PascalCase (e.g., `Project.Folder.Subfolder`)
- Properties: PascalCase (e.g., `MyProperty`)
- Private fields: camelCase with an underscore prefix (e.g., `_myPrivateField`)
- Internal fields: camelCase with an underscore prefix (e.g., `_myInternalField`)
- Public fields: PascalCase (e.g., `MyPublicField`)
- Static fields: PascalCase (e.g., `MyStaticField`)
- Private or Internal Static fields: camelCase with the "s_" prefix (e.g., `s_myStaticField`)
- Tuples: camelCase (e.g., `(int myInt, string myString)`)
- Local functions: camelCase (e.g., `myLocalFunction()`)
- Parameters: camelCase (e.g., `myParameter`)
- Methods: PascalCase (e.g., `MyMethod()`)
- Private methods: PascalCase
- Internal methods: PascalCase
- Public methods: PascalCase
- Static methods: PascalCase
- Private or Internal Static methods: PascalCase
- Async methods: PascalCase with an "Async" suffix (e.g., `MyMethodAsync()`)
- Record types: PascalCase (e.g., `MyRecord`)
- Delegates: PascalCase (e.g., `MyDelegate`)
-Structs: PascalCase (e.g., `MyStruct`)
- Generic type parameters: PascalCase (e.g., `T`, `TKey`, `TValue`)


NODE.JS naming conventions should be followed for all code in this repository. This includes:
- Variables: camelCase (e.g., `myVariable`)
- Constants: UPPER_SNAKE_CASE (e.g., `MY_CONSTANT`)
- Functions: camelCase (e.g., `myFunction()`)
- Classes: PascalCase (e.g., `MyClass`)
- Modules: PascalCase (e.g., `MyModule`)
- Packages: PascalCase (e.g., `MyPackage`)
